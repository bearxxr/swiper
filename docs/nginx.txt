Nginx在项目中的作用
    1.负载均衡
        轮询
        权重
        最小连接数
            根据nginx服务器的多少分配任务
            当某一台任务少二零一台任务多时，任务优先给任务量少的服务器
        ip hash
            用户的不同ip地址
                nginx server 0
                nginx server 1   ip1,ip3
                nginx server 2   ip2

                ip1 = '192.168.7.11'  %3 = 2
                ip2 = '192.168.1.666' %3 = 1
                ip3 = '192.168.3.99'  %3 = 1
            用户访问服务端，nginx会把不同的ip地址进行一个取余操作，结果会成为 0,1,2等
            然后各自对应的服务会处理这些对应的ip地址
    2.分发路由(反向代理)
        一般来说，用户启动服务的流程如下：
                               / Flask服务
            user --> proxy ----  Django服务
                               \ Tornado服务
            用户操作，然后nginx会代理上述的各种服务，把服务的结果返回给用户。
            就是把服务器的结果给用户发送，减少服务器的负担。
    3.加载静态文件
        就是处理照片，文档，视频等。
